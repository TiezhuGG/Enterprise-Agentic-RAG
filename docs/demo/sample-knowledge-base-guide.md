# 企业知识库使用规范

## 知识空间角色

企业知识库使用 Knowledge Space 管理资料边界。每个知识空间至少有一个 OWNER。

- OWNER 负责空间治理、成员管理、资料删除和重要配置。
- EDITOR 可以上传、更新和重新处理文档。
- VIEWER 只能读取已经完成处理的文档内容。

## 文档入库要求

文档上传后需要经过解析、清洗、分块、向量化和索引同步。只有状态为 READY 的文档才应进入正式检索结果。

如果文档处理失败，维护人员应先查看 Pipeline Timeline 中的失败阶段，再决定是否重新 Ingest。

## 引用可信度

AI 回答必须优先基于知识库检索结果。回答中展示的 Citation 应包含来源文档、章节标题、相关片段和相关度分数。

如果没有找到依据，助手应明确说明“没有找到足够依据”，而不是编造制度条款。

## 敏感信息管理

包含个人身份信息、财务凭证、客户合同或内部战略的资料，应设置为 INTERNAL 或 CONFIDENTIAL。

CONFIDENTIAL 文档不得在跨部门检索中默认开放，只允许空间 OWNER、被授权部门或具备保密知识读取权限的用户读取。
